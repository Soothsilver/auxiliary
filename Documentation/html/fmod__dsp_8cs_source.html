469  Normalize amplifies the sound based on the maximum peaks within the signal.<br>
470  For example if the maximum peaks in the signal were 50% of the bandwidth, it would scale the whole sound by 2.<br>

524  /*
525  [ENUM]
526  [
527  [DESCRIPTION]
528  Parameter types for the FMOD_DSP_TYPE_PITCHSHIFT filter.
529 
530  [REMARKS]
531  This pitch shifting unit can be used to change the pitch of a sound without speeding it up or slowing it down.<br>
532  It can also be used for time stretching or scaling, for example if the pitch was doubled, and the frequency of the sound was halved, the pitch of the sound would sound correct but it would be twice as slow.<br>
533  <br>
534  <b>Warning!</b> This filter is very computationally expensive! Similar to a vocoder, it requires several overlapping FFT and IFFT's to produce smooth output, and can require around 440mhz for 1 stereo 48khz signal using the default settings.<br>
535  Reducing the signal to mono will half the cpu usage, as will the overlap count.<br>
536  Reducing this will lower audio quality, but what settings to use are largely dependant on the sound being played. A noisy polyphonic signal will need higher overlap and fft size compared to a speaking voice for example.<br>
537  <br>
538  This pitch shifter is based on the pitch shifter code at http://www.dspdimension.com, written by Stephan M. Bernsee.<br>
539  The original code is COPYRIGHT 1999-2003 Stephan M. Bernsee <smb@dspdimension.com>.<br>
540  <br>
541  '<i>maxchannels</i>' dictates the amount of memory allocated. By default, the maxchannels value is 0. If FMOD is set to stereo, the pitch shift unit will allocate enough memory for 2 channels. If it is 5.1, it will allocate enough memory for a 6 channel pitch shift, etc.<br>
542  If the pitch shift effect is only ever applied to the global mix (ie it was added with System::addDSP), then 0 is the value to set as it will be enough to handle all speaker modes.<br>
543  When the pitch shift is added to a channel (ie Channel::addDSP) then the channel count that comes in could be anything from 1 to 8 possibly. It is only in this case where you might want to increase the channel count above the output's channel count.<br>
544  If a channel pitch shift is set to a lower number than the sound's channel count that is coming in, it will not pitch shift the sound.<br>
545 
546  [PLATFORMS]
